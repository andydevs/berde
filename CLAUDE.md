## Project overview

berde is a migration system for Serde data files (e.g. versioned JSON/YAML/TOML configs that need to be
upgraded between schema versions). The project is in early scaffolding stages — the core library
(`crates/berde`) currently contains only placeholder code generated by `cargo new`.

## Workspace layout

- This is a Cargo workspace (`Cargo.toml` at the root) with an explicit `members` list (no glob).
- `crates/berde` — the main library crate (package name `berde`), part of the workspace.
- `demos/json_schema` — a demo crate, part of the workspace. It has `publish = false` set, since demo
  crates are not meant to be published to crates.io.

When adding a new crate under `crates/` or `demos/`, add its path to the root `Cargo.toml`
`members` list explicitly. Demo crates should set `publish = false`.

### `demos/json_schema`

This demo shows the versioned-schema layout that `berde` is being designed around:

- Each schema version is its own module under `src/versions/`, named `vN.rs` (e.g. `v1.rs`,
  `v2.rs`), and defines a `Schema` type for that version's data layout.
- `build.rs` scans `src/versions/` at build time, sorts the `vN.rs` files numerically by `N`,
  and generates `pub mod vN;` declarations into `$OUT_DIR/versions.rs`.
- `src/lib.rs` includes that generated file inside `pub mod versions { ... }`, exposing each
  version as `versions::v1`, `versions::v2`, etc.
- `build.rs` also generates `$OUT_DIR/latest.rs`, containing `pub use versions::vN as latest;`
  for the highest-numbered version. `src/lib.rs` includes this at the crate root, so the most
  recent schema is always available as `latest::Schema`.

To add a new schema version, add a new `vN.rs` file under `src/versions/` — no changes to
`src/lib.rs` are needed, and `latest` automatically points to the new version.

## Common commands

Run from the repo root (operates on the whole workspace, including demos):
- `cargo build` — build all workspace crates
- `cargo test` — run all tests in the workspace
- `cargo test -p berde` — run tests for just the `berde` crate
- `cargo test -p json_schema` — run tests for just the demo crate
- `cargo test <test_name>` — run a single test by name (e.g. `cargo test it_works`)
- `cargo check` — type-check without building
- `cargo fmt` — format code
- `cargo clippy` — lint

## Edition

All crates target Rust 2024 edition.
